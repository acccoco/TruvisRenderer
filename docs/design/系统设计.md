## 依赖传递

`BindlessManager` 需要根据 frame id 来进行 dirty 标记，slot 管理。

frame id 是 `Renderer` 管理的字段，如何将这个字段传到到 `BindlessManager`，我想到了这些方式：

1. 将 `FrameCounter` 作为全局变量，`BindlessManager` 直接访问。但是这样会导致隐式依赖
2. 在 `BindlessManager` 内部维护一个 frame id 的字段，在 begin frame 的时候由外部传入。但是这样语义错误
3. 在 `BindlessManager` 内部维护 `FrameCounter` 的引用，这样可能导致借用检查灾难
4. 通过 register 接口，将 frame id 参数传入。这样的接口设计比较繁琐

最终选择的方案是：新增一个语义化的类型 `FrameToken(u64)`，在 `BindlessManager` 内部维护一个 `FrameToken` 的字段，
在 begin frame 的时候由外部传入。这样既避免了隐式依赖，又保持了接口的简洁性，同时也不会导致借用检查灾难。
